# Contributing to Business Central Apps

Thank you for your interest in contributing to our Business Central extensions! We welcome contributions from the community.

## How to Contribute

### Reporting Issues

- Check if the issue already exists in the [Issues](https://github.com/walter75-Consulting/Business-Central-Apps/issues) section
- Provide a clear description of the problem
- Include steps to reproduce the issue
- Specify the Business Central version you're using
- Add relevant error messages or screenshots

### Suggesting Features

- Open a new issue with the label "enhancement"
- Describe the feature and its benefits
- Explain use cases where this feature would be helpful

### Pull Requests

1. **Fork the repository** and create your branch from `main`
2. **Follow AL coding standards** and Business Central best practices
3. **Test your changes** thoroughly in a Business Central environment
4. **Update documentation** if you're changing functionality
5. **Ensure your code compiles** without errors or warnings
6. **Submit a pull request** with a clear description of the changes

### Development Setup

1. Clone the repository
2. Install AL Language extension for VS Code
3. Configure your AL-Go settings as needed
4. Use the provided AL-Go workflows for building and testing

### Coding Guidelines

- Follow Microsoft's AL coding guidelines
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and concise
- Maintain consistent indentation and formatting

### App-Specific Guidelines

Each app folder may contain a `TECHNICAL_DOCUMENTATION.md` with specific implementation details and guidelines.

## Code of Conduct

Please be respectful and constructive in all interactions. We aim to foster an inclusive and welcoming community.

## Questions?

If you have questions about contributing, feel free to open an issue or contact us through our website.
